Collections​

Collection คือหมวดหมู่ของเนื้อหาต่าง ๆ หรืออีกชื่อนึงอาจจะเรียกว่า "product" ต่าง ๆ ของ PIER นั่นเอง ประกอบไปด้วย

  • บทความ
    • Discussion Paper บทความเชิงวิชาการ ในรูปแบบ working paper เป็นภาษาอังกฤษ
    • PIERspectives ยทความแบบสังเคราะห์ รวบรวมเนื้อหาของ reseasrch paper ที่นักวิจัยคนอื่น ๆ ทำไว้ เป็นภาษาไทย
    • aBRIDGEd บทความแบบสั้น กระชับ เน้นให้เข้าใจง่ายสำหรับผู้ออกนโยบาย
    • เศรษฐศาสตร์เข้า "ท่า" บทความลงคอลัมน์ของ ThaiPublica ที่สั้น กระชับกว่า aBRIDGEd
    • Blog งานเขียนที่สั้นมาก ๆ มีใจความสำคัญไม่ซับซ้อน
    • Announcements ข่าว ประกาศ ของ PIER
  • Events
    • Conferences งานประชุมเชิงวิชาการ หลัก ๆ จะเป็น BOT Symposium แต่อาจจะมี conference อื่น เช่น งานที่จัดร่วมกับ NBER เป็นต้น
    • Research Workshops งานประชุมเชิงวิชาการที่จัดขึ้นเพื่อให้นักวิชาการมา present งานวิจัยของตัวเอง
    • Policy Forums งานประชุมเชิงนโยบาย ที่เน้นการพูดคุยกับผู้กำหนดนโยบายต่าง ๆ
    • Seminars การบรรยายเชิงวิชาการที่เชิญผู้เขี่ยวชาญในด้านนั้น ๆ มาบรรยายงานวิจัย
    • Exchanges การบรรยายเชิงวิชาการที่เชิญนักวิจัยมา present งานวิจัยเพื่อรับ feedback ไปพัฒนางานวิจัยต่อ
  • อื่น ๆ
    • Members หน้า profile ของสมาชิก PRN
    • Pages หน้าอื่น ๆ

Templates​

การแสดงผลในเว็บไซต์ นอกจากเนื้อหาแล้ว ยังต้องมีข้อมูลอื่น ๆ เกี่ยวกับบทความหรือ event นั้น ๆ เช่น เช่น ชื่อบทความ ชื่อผู้เขียน รวมถึงรูปแบบการแสดงผลในเว็บไซต์ด้วย

เราสามารถกำหนดข้อมูลเหล่านี้ไว้ในส่วนที่เรียกว่า frontmatter ซึ่งถูกกำหนดโดยขีดสามขีดด้านบนและล่าง จากตัวอย่างด้านล่าง เรารู้ว่า id ของบทความนี้ คือ 01 และชื่อของบทความ คือ บทความแรก เป็นต้น

---
id: "01"
title: บทความแรก
...
---
เนื้อหาของไฟล์นี้...

ตัว frontmatter นี้ ใช้ syntax ของ YAML

บทความหรือ event ในแต่ละ collection ก็จะใช้ template แตกต่างกันไป

  • งานเขียนทั้งหมด ใช้ template แบบ publication
  • Events ประเภท conference หรือ workshop ใช้ template แบบ conference นอกจากนั้นใช้แบบ seminar เว้นแต่ว่าจะกำหนดไว้ว่าจะใช้ template แบบไหนไว้ใน frontmatter
  • Members ใช้ template แบบ member
  • Pages ใช้ template แบบ page

รายละเอียด attribute ต่าง ๆ​

ทั่วไป​

id​

หมายเลขประจำบทความหรือ event

urlname​

URL แบบสั้น ที่สื่อความหมายมากขึ้น จะปรากฎในที่อยู่ของเนื้อหานั้น

title​

ชื่อบทความ หรือชื่อสมาชิก PRN

subtitle​

Subtitle ของบทความ aBIRDGEd จะเป็นประโยคสั้น ๆ ไม่เกิน 1 บรรทัด

series​

Series ของบทความนั้น

excerptOverride​

ตามปกติ excerpt ที่แสดงในหน้าแรกจะตัดเอาส่วนแรกของบทความมา ถ้าส่วนแรกเป็นส่วนที่เราไม่ต้องการ (เช่น เป็น html element) ใช้ส่วนนี้มา override ได้

authors​

รายชื่อผู้เขียน (ตาม memberid)

date​

วันที่เผยแพร่บทความครั้งแรก รูปแบบ yyyy-mm-dd

updated​

วันที่ปรับปรุงบทความล่าสุด

topics​

รายชื่อ topic ที่เกี่ยวข้อง

tags​

รายชื่อ tag ที่เกี่ยวข้อง

lang​

กรณีที่บทความนั้นไม่ได้ใช้ภาษาปกติ (ไทย ยกเว้น DP เป็นอังกฤษ) ใส่ list ของภาษา (th หรือ en) ตรงนี้

cover​

ภาพประกอบบทความ (ดูคำแนะนำที่นี่)

hideCover​

ตั้งให้เป็น true เพื่อไม่โชว์ cover ในหน้าบทความ (รูป cover จะปรากฏในหน้า list บทความเท่านั้น)

coverCredits​

เครดิตภาพประกอบบทความ กรณีที่ไม่ได้ใช้รูปจาก ShutterStock ใส่เป็นแบบ markdown ได้

thumbnail​

รูป thumbnail มีประโยชน์กรณีที่รูป cover มีรายละเอียดเยอะเกินไป แสดงในหน้า list บทความแทนรูป cover

endLinks​

{label, file} shown at bottom of body (before | xxdisclaimer

layout​

ปรับ layout ของหน้า ตอนนี้ถ้าจะกำหนดทำได้อย่างเดียวคือเป็น onecol ซึ่งจะเอารูปผู้เขียนไปไว้ด้านล่าง เพื่อให้เนื้อหาได้เนื้อที่กว้างเต็มหน้า

bibliography​

สำหรับบทความที่มีแหล่งอ้างอิง (โดยทั่วไปคือ aBRIDGEd) ใส่ชื่อไฟล์ BiBTeX ไว้

featuredTop​

สำหรับ content ที่ต้องการนำเสนอใน main slider ในหน้าแรก

สำหรับ Discussion Paper​

fullText​

ชื่อไฟล์ full text PDF

jel​

รายการของ JEL classification

length​

ความยาว (ไม่รวมหน้าปก) ของ full text PDF สำหรับ Discussion Paper สำหรับใส่ใน RePEc

published​

สำหรับ Disucussion Paper ที่ได้รับการตีพิมพ์แล้ว

สำหรับ Event​

discussants​

รายชื่อผู้วิจารณ์ (ตาม memberid)

endDate​

วันสิ้นสุดของ conference กรณี conference ที่จัดหลายวัน

location​

สถานที่จัดงาน

locationUrl​

ลิงก์ของสถานที่จัดงาน (เช่น แผนที่)

liveUrl​

ลิงก์ของ live event

data​

ข้อมูล event ต่าง ๆ ของ conference (ดูรายละเอียดเพิ่มเติม)

details​

ในกรณีที่อยากจัดรูปแบบของรายละเอียดใน data (ดูรายละเอียดเพิ่มเติม)

notime​

ตั้งเป็น true ถ้า data ด้านบนไม่มีข้อมูลเวลาอยู่

startTime​

เวลาเริ่ม seminar

endTime​

เวลาจบ seminar

video​

ลิงก์ไปยังวีดีโอของ event นั้น

program​

ลิงก์ไปยังสูจิบัตรของงาน

template​

กำหนด template กรณีที่ไม่ได้ใช้ template มาตรฐาน (เช่น เป็น event แบบ forum แต่ต้องการใช้ template แบบ conference)

hideAuthors​

กำหนดให้เป็น true ถ้าไม่ต้องการให้แสดงชื่อผู้เขียนในหน้า list (ใช้กรณีที่ authors เป็นคนเขียนสรุป ไม่ได้เป็นคน present ในงานนั้น เป็นต้น)

endNote​

หมายเหตุด้านล่าง เช่น conference นี้จัดร่วมกับใคร

showDisclaimer​

ตามปกติถ้าไม่มี authors จะไม่มี disclaimer ด้านล่าง แต่กรณีที่ตั้งใจไม่ใส่ authors (เช่น policy forums) แต่อยากให้มี disclaimer ขึ้น สามารถตั้งค่าเป็น true ได้

featuredEvent​

สำหรับกำหนด featured event ที่ปรากฏในหน้าแรก

สำหรับ Member​

memberid​

ชื่อภาษาอังกฤษ ใช้เป็น key ในการเชื่อมโยงกับบทความต่าง ๆ

org​

องค์กรที่สมาชิกสังกัด (ภาษาตามไฟล์) ถ้าเป็นองค์กรตาม list ด้านล่าง สามารถใช้ตัวย่อได้เลย

image​

ชื่อไฟล์รูปภาพของสมาชิก

fields​

ประเภทงานวิจัยที่สนใจ

jobTitle​

ตำแหน่ง (ปรากฏในหน้า staff เท่านั้น)

jobTitleValue​

ลำดับของตำแหน่ง โดย 9.99 = ที่ปรึกษา, 9.x = ตำแหน่งวิจัย, 8.x = ตำแหน่งนักวิเคราะห์, 7.x = ตำแหน่งสนับสนุน และ x คือระดับชั้น (เช่น นักวิจัยระดับ ผบส. เป็น 9.7)

featuredEconomist​

สำหรับกำหนด featured economist ที่ปรากฎในหน้าแรก

exclude​

ถ้าเป็น true ให้ไม่สร้างหน้า profile นั้นขึ้นมา ใช้สำหรับคนที่ปรากฏตัวบ่อย ๆ แต่ไม่ได้เป็น PRN

email​

อีเมลของสมาชิก

website​

ลิงก์ไปเว็บไซต์ส่วนตัวของสมาชิก

facebook​

ลิงก์ไป Facebook profile ของสมาชิก

twitter​

ลิงก์ไป Twitter profile ของสมาชิก

linkedin​

ลิงก์ไปหน้า LinkedIn ของสมาชิก

github​

ลิงก์ไป GitHub profile ของสมาชิก

firstVisit​

ปี ค.ศ. ที่สมาชิกมาเป็น Visiting Fellow ครั้งแรก

currentFellow​

สมาชิกคนนั้นเป็น Visiting Fellow อยู่ในขณะนี้มั้ย

prnOfficer​

สมาชิกคนนั้นเป็น PRN Officer มั้ย หลัง ๆ มาไม่ค่อยได้ใช้แล้ว

oldId​

ID จากเว็บไซต์เดิมแบบ Wordpress สำหรับการ redirect
